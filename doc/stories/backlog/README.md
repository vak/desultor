# backlog/

Backlog stories are real initiatives that matter, but are not the current
focus.

Backlog also plays the parking role:

- accepted work can wait here honestly;
- state and reason belong in `STATUS.md`;
- a separate top-level parked bucket is not required.

Backlog is not the only story entrypoint. Blocking or urgent issues may bypass
backlog and start directly in `active/`.

This is the stable parking mechanism defined by
`doc/spec/desultor/inbox-triage-and-parking-v0_1.md`.

If starter-owned backlog should stay clearly detachable, group it under
`backlog/desultor/`.

# Orchestrator replacement bridge

## Why this story exists

The starter is explicitly temporary. If a stronger orchestrator appears later,
migration should not require rethinking the whole repository from scratch.

## Desired outcome

- explicit list of what must survive replacement;
- explicit list of what can disappear cleanly;
- a migration note from Desultor conventions to a real orchestrator.

## Likely artifacts

- RFC or spec clarifying replacement boundaries
- migration checklist
- cleanup guidance for obsolete starter files

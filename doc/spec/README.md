# spec/

`spec/` holds stable contracts for how the repository is organized and how work
should move from rough exploration to reviewed delivery.

Use this directory for things the project is willing to treat as operational
defaults. If an idea is still broad, unstable, or full of open branches, it
belongs in `doc/rfc/` first.

If the repository is also a reusable starter, keep starter-owned contracts in
`spec/desultor/` so host-project contracts can later live at the root of
`spec/` without ambiguity.

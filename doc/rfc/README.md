# rfc/

`rfc/` is the place for broad seeds, open design questions, and option
comparison before a decision is stable enough for `doc/spec/`.

If a document is still too broad, too unstable, or too full of unresolved
branches, it belongs here rather than in `spec/`.

Starter-owned RFC material may be grouped under `rfc/desultor/` so that host
project RFCs can later use the root `rfc/` namespace cleanly.
